## detailed batch tasks

### iteration 1 – docs consolidation

1.1 deduplicate md files via ps1 script (dry run)  
1.2 move canonical files into new sub-dirs  
1.3 update x-refs, regenerate nav  
1.4 run mkdocs link-checker plugin – fail build on error

### iteration 2 – import optimisation

2.1 generate import graph with pyan3  
2.2 identify modules >1 MB loaded at startup  
2.3 move to deferred/conditional import pattern and test

### iteration 3 – database gap patch

3.1 diff schema vs scripts, generate alembic revision  
3.2 add notifications, api_keys, audit tables  
3.3 integration tests with sqlite & postgres fixtures

### iteration 4 – responsive templates

4.1 audit templates for fixed widths  
4.2 apply tailwind container & clamp sizing  
4.3 dark mode color tokens using tailwind palette  
4.4 lighthouse / pa11y run

### iteration 5 – automated tests

5.1 add cypress ui tests @ 320,1440,8k  
5.2 locust perf script 100 rps / 5 min  
5.3 bandit + safety security scan in ci

### iteration 6 – async & lazy module loading

6.1 background thread model fetch with progress api  
6.2 importlib entry-point plugin loader for optional providers

### iteration 7 – deployment & speed

7.1 gunicorn conf workers=cpu*2+1, preload off  
7.2 brotli compress static assets post-build  
7.3 nginx cache-control immutables, hsts preload

### iteration 8 – final audit

8.1 dupfinder run on repo  
8.2 mkdocs build with strict mode  
8.3 tag release v2024-08-rc1

## risk & mitigation

* **schema migration failure** – run pg_dump before alembic upgrade, test on staging clone.  
* **lazy import regressions** – contract tests on critical routes, enable sentry in staging.  
* **template breakage** – visual regression snapshots in cypress.
