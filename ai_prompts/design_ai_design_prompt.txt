## Overview
This prompt defines the design system for AI Valido Online, focusing on a responsive, accessible, and performant frontend. It includes a chat interface for AI-driven financial queries, UI components (tables, cards, menus, charts), and a template system, all built with Tailwind CSS and component libraries (Flowbite, DaisyUI, Pagedone, TailGrids, Preline). Examples demonstrate best practices for lazy loading, asset optimization, caching, responsive images, and a mobile-first approach with 2xl/3xl breakpoints. The system ensures a consistent design language, modular components, and accessibility (WCAG 2.1 compliance).

## General Guidelines
- **Framework**: Use Tailwind CSS for styling, with Flowbite, DaisyUI, Pagedone, TailGrids, and Preline for prebuilt components.
- **Responsive Design**: Mobile-first with breakpoints (sm, md, lg, xl, 2xl, 3xl) for screens up to 2560px.
- **Accessibility**: Follow WCAG 2.1 (ARIA labels, keyboard navigation, high-contrast modes).
- **Performance**:
  - Lazy load components/images using `loading="lazy"`.
  - Minify CSS/JS with tools like `esbuild` or `terser`.
  - Implement browser caching (Cache-Control headers) and CDN caching (e.g., Cloudflare).
  - Use responsive images with `srcset` and `sizes`.
- **Modularity**: Create reusable components as Tailwind utility classes or custom components in component libraries.
- **Iteration Strategy**:
  - **Iteration 1 (Days 1-2)**: Implement chat interface and basic UI components (Flowbite tables, DaisyUI cards).
  - **Iteration 2 (Days 3-4)**: Add mega menu, charts (ApexCharts), and performance optimizations (lazy loading, minification).
  - **Iteration 3 (Days 5-6)**: Finalize template system, caching, and accessibility enhancements.

## Task 2: Frontend Performance
- **Objective**: Optimize frontend performance with lazy loading, asset optimization, caching, and responsive images.
- **Components**:
  - Lazy loading: Use `loading="lazy"` for images, `IntersectionObserver` for components.
  - Asset optimization: Minify CSS/JS with `esbuild`, bundle with `vite`.
  - Caching: Set `Cache-Control: max-age=31536000` for static assets, use CDN (Cloudflare).
  - Responsive images: Use `srcset` and `sizes` for images.
- **Prompt**:
  ```
  Task: Generate performance-optimized frontend code.
  Input: Component: {component}, Features: Lazy loading, Asset optimization, Caching, Responsive images
  Instructions:
  1. Create a component (e.g., Flowbite table, ApexCharts chart) with lazy loading (`IntersectionObserver`).
  2. Use minified CSS/JS CDNs (Tailwind, Flowbite, ApexCharts).
  3. Add caching headers in HTML meta or server config.
  4. Include responsive images with srcset/sizes.
  5. Return HTML with scripts.
  Example Output:
  <div class="lazy-load" data-component="table">
    <table class="min-w-full divide-y divide-gray-200 hidden">
      <thead><tr><th class="px-6 py-3">Invoice Number</th><th class="px-6 py-3">Amount</th></tr></thead>
      <tbody><tr><td class="px-6 py-4">INV001</td><td class="px-6 py-4">12000</td></tr></tbody>
    </table>
  </div>
  <img src="low-res.jpg" srcset="high-res.jpg 2x, med-res.jpg 1.5x" sizes="(max-width: 600px) 100vw, 50vw" loading="lazy" alt="Financial Chart">
  <meta http-equiv="Cache-Control" content="max-age=31536000">
  <script>
    const observer = new IntersectionObserver((entries) => {
      entries.forEach(entry => {
        if (entry.isIntersecting) {
          entry.target.querySelector('table').classList.remove('hidden');
          observer.unobserve(entry.target);
        }
      });
    });
    document.querySelectorAll('.lazy-load').forEach(el => observer.observe(el));
  </script>
  ```
- **Implementation**:
  ```html
  <!DOCTYPE html>
  <html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta http-equiv="Cache-Control" content="max-age=31536000">
    <title>Optimized Table</title>
    <link href="https://cdn.jsdelivr.net/npm/tailwindcss@2.2.19/dist/tailwind.min.css" rel="stylesheet">
    <link href="https://cdn.jsdelivr.net/npm/flowbite@1.6.5/dist/flowbite.min.css" rel="stylesheet">
  </head>
  <body class="bg-gray-100">
    <div class="lazy-load max-w-4xl mx-auto p-4" data-component="table">
      <table class="min-w-full divide-y divide-gray-200 hidden">
        <thead class="bg-gray-50">
          <tr>
            <th class="px-6 py-3 text-left text-xs font-medium text-gray-500 uppercase" scope="col">Invoice Number</th>
            <th class="px-6 py-3 text-left text-xs font-medium text-gray-500 uppercase" scope="col">Amount</th>
          </tr>
        </thead>
        <tbody class="bg-white">
          <tr>
            <td class="px-6 py-4">INV001</td>
            <td class="px-6 py-4">12000 RSD</td>
          </tr>
        </tbody>
      </table>
    </div>
    <img src="/images/chart-low.jpg" srcset="/images/chart-high.jpg 2x, /images/chart-med.jpg 1.5x" sizes="(max-width: 600px) 100vw, 50vw" loading="lazy" alt="Financial summary chart" class="mx-auto mt-4">
    <script src="https://cdn.jsdelivr.net/npm/flowbite@1.6.5/dist/flowbite.min.js" defer></script>
    <script>
      const observer = new IntersectionObserver((entries) => {
        entries.forEach(entry => {
          if (entry.isIntersecting) {
            entry.target.querySelector('table').classList.remove('hidden');
            observer.unobserve(entry.target);
          }
        });
      }, { rootMargin: '100px' });
      document.querySelectorAll('.lazy-load').forEach(el => observer.observe(el));
    </script>
  </body>
  </html>
  ```

  ```bash
  # Minify with esbuild
  esbuild styles.css --minify --outfile=styles.min.css
  esbuild script.js --minify --outfile=script.min.js
  ```
- **Best Practices**:
  - Use `defer` for scripts to avoid render-blocking.
  - Set `rootMargin` in `IntersectionObserver` for smooth lazy loading.
  - Use CDN for minified assets (e.g., Tailwind, Flowbite).
  - Configure server for caching: `Cache-Control: max-age=31536000` for static assets.

  - Use a single layout file with `<slot>` or dynamic includes for content.
  - Apply Tailwind’s `min-h-screen` for full-height layouts.
  - Lazy load main content with `IntersectionObserver`.
  - Use `aria-label` for navigation links and `hover` states for interactivity.

  ```python
  from flask import Flask, render_template
  app = Flask(__name__)
  ```
